# Sprint Release Notes Agent

**Problem**: Writing release notes manually takes 4–6 hours every sprint and always misses details.

**Solution**: Agent reads all closed Jira tickets → auto-generates perfect, stakeholder-ready release notes in under 60 seconds.

**Features included**:
- Customer-facing summary
- Technical changes
- Bug fixes
- Known issues
- Migration notes

Used in production for last 4 sprints → saved ~20 hours/month.
